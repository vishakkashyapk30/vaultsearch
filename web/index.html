The problem

Retrieval-augmented generation puts a language model in front of a company’s documents. The moment you do that, you inherit an access-control problem: the model must never build an answer — or a citation, or even a hint — out of a document the person asking is not allowed to read. A single leaked sentence is a breach.

The approach

  1. Filter before you search. The user is expanded to their identity and group principals, and both keyword (BM25) and vector search are restricted to authorized documents before any scoring happens. Restricted text never enters ranking, reranking, or the prompt.
  2. Verify again before you answer. An independent check re-confirms every retrieved chunk against the user’s permissions — defense in depth, in case retrieval ever regresses.
  3. Sanitize the output. Any citation the model invents that doesn’t map to authorized evidence is stripped before the response leaves the server.

The authorization decision is always deterministic code. The language model plans searches and phrases answers; it is never trusted to decide who can see what.
